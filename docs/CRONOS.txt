\section sec_CRONOS_install What do I need to run CRONOS?

CRONOS is released as open source code under the <A href="http://www.eclipse.org/legal/epl-v10.html">Eclipse Public License (EPL)</A>.

CRONOS depends on the following third-party libraries:
- <A href="https://projects.coin-or.org/MCpp">MC++</A> (version 2.0) for construction, manipulation and bounding of factorable functions;
- <A href="https://projects.coin-or.org/Ipopt">IPOPT</A> for local solution of NLP problems;
- <A href="http://www.gurobi.com/">GUROBI</A> and/or <A href="http://www-01.ibm.com/software/commerce/optimization/cplex-optimizer/">CPLEX</A> for solution of LP/MILP problems;
- <A href="https://computation.llnl.gov/casc/sundials/main.html">SUNDIALS</A> and/or <A href="http://www.gnu.org/software/gsl/">GSL</A> for non-validated integration of ODEs/DAEs
.

On distribution, the main directory should contain the files <TT>AUTHORS</TT>, <TT>CHANGELOG</TT>, <TT>INSTALL</TT>, <TT>LICENSE</TT> and <TT>README</TT>, as well as 3 subdirectories:
- <tt>src</tt> contains the source code of CRONOS, which consists of header files only; that is, there is no need to build or link a library.
- <tt>doc</tt> contains the Doxygen documentation for CRONOS, which can be accessed by opening the file <tt>doc/html/index.html</tt> with your favorite browser.
- <tt>test</tt> contains simple examples that should help you understand how to use the types of CRONOS.
.

\section sec_MC_examples How do I run the test examples of CRONOS?

Change into the <tt>test</tt> directory, and select one of the subdirectories, e.g.

<tt>
$ cd test/NLGO
</tt>

At the command line, create the executables by typing:

<tt>
$ make
</tt>
