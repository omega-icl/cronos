CRONOS provides a collection of C++ classes for global optimization and constraint propagation of nonlinear and dynamic systems using complete search.

What do I need to run CRONOS?

CRONOS is released as open source code under the Eclipse Public License (EPL).

CRONOS depends on the following third-party libraries:

  • MC++ (version 2.0) for construction, manipulation and bounding of factorable functions;
  • IPOPT for local solution of NLP problems;
  • GUROBI and/or CPLEX for solution of LP/MILP problems;
  • SUNDIALS and/or GSL for non-validated integration of ODEs/DAEs

On distribution, the main directory should contain the files AUTHORS, CHANGELOG, INSTALL, LICENSE and README, as well as 3 subdirectories:

  • src contains the source code of CRONOS, which consists of header files only; that is, there is no need to build or link a library.
  • doc contains the Doxygen documentation for CRONOS, which can be accessed by opening the file doc/html/index.html with your favorite browser.
  • test contains simple examples that should help you understand how to use the types of CRONOS.
CRONOS has been developed and tested primarily under Linux
The tests and examples should compile without warnings with gcc (ver. 4.7) after enabling the C++11 standard features

Where can I learn more about CRONOS?

How do I run the test examples of CRONOS?

Change into the test directory, and select one of the subdirectories, e.g.

$ cd test/NLGO

At the command line, create the executables by typing:

$ make
